Toàn bộ project chia làm 3 lớp rõ ràng: Data layer (sinh và xử lý dữ liệu), Runtime layer (agents chạy thật), Infrastructure layer (config, test, deploy). Ba lớp này độc lập — có thể train lại model mà không đụng runtime, có thể swap agent mà không ảnh hưởng training.

data/
Chứa toàn bộ dữ liệu, tách thành 4 sub-folder:

raw/ — dữ liệu gốc nếu có (menu thật, FAQ thật)
synthetic/ — output của pipeline A1.2, các file JSON 4000+ samples
processed/ — đã split train/val/test, đã cân bằng 4 class
checkpoints/ — trạng thái pipeline để resume nếu crash giữa chừng

menu.json đặt ở đây luôn vì cả Order Agent lẫn Consultant Agent đều cần đọc nó.

src/router/
Đây là code của module A1.1 và A1.3:

model.py — load SLM (Qwen2.5-1.5B hoặc nhỏ hơn), quản lý vòng đời model, warm-up
classifier.py — nhận text đầu vào, chạy inference, trả {"action": "<intent>"} chuẩn JSON
prompts.py — system prompt cho router, tách ra file riêng để dễ tune
benchmark.py — đo latency, tính confusion matrix, F1-score — dùng cho tiêu chí nghiệm thu


src/pipeline/
Đây là code của module A1.2 — data generation pipeline:

generator.py — gọi LLM async để sinh samples, mỗi batch ghi checkpoint
deduplicator.py — hash từng sample, loại trùng trước khi lưu
tracker.py — progress bar, log số lượng theo intent, resume từ checkpoint
validator.py — kiểm tra schema JSON output đúng format trước khi lưu vào data/synthetic/


src/agents/
Đây là code của module A2.1:

base.py — abstract class BaseAgent định nghĩa interface chung: async def handle(query, session). Mọi agent kế thừa từ đây, đảm bảo consistency.
order_agent.py, consultant_agent.py, faq_agent.py — mỗi file là 1 specialized agent, implement logic riêng
prompts/ — sub-folder chứa system prompt của từng agent tách riêng, dễ chỉnh mà không cần đọc code


src/session/
Module A2.2:

store.py — SessionStore class, dùng dict hoặc Redis, lưu history theo session_id, enforce window 5 turns
summarizer.py — khi history vượt 70% context window thì gọi LLM tóm tắt phần cũ
cleanup.py — background async task chạy mỗi vài phút, xóa session hết TTL 30 phút


src/queue/
Module A2.3:

request_queue.py — asyncio.Semaphore giới hạn concurrent LLM calls, FIFO với timeout 60s
retry.py — decorator @retry_with_backoff dùng cho mọi API call trong project, delay tăng dần 1s → 2s → 4s


training/
Tách hoàn toàn khỏi src/ vì đây là code chạy một lần trên cloud, không phải serving:

sft_train.py — fine-tune SFT trên H100, load data từ data/processed/
quantize.py — export AWQ/GPTQ để serve trên RTX 3060, và GGUF cho edge
evaluate.py — sinh confusion matrix, per-class F1, phân tích hard samples


tests/
Mỗi file test map 1-1 với 1 module chính. Chạy pytest tests/ là kiểm tra toàn bộ hệ thống trước khi demo.

scripts/
run_pipeline.py chạy gen data từ openAI (chạy A1.2)
run_training.py chạy fine turning (A1.3)

Root files

config.yaml — tất cả hyperparameter, model path, timeout, semaphore limit đều ở đây, không hardcode trong code
main.py — FastAPI entrypoint, import router + agents, expose /chat endpoint
docker-compose.yml — spin up toàn bộ stack một lệnh
Makefile — shortcut: make generate-data, make train, make serve — tiện khi demo
notebooks/ — EDA, demo interactive, không ảnh hưởng production code
